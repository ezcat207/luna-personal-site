### 2. 教学三原则 (Theory Section)

#### 原则 1: 感性先行 (Vibe First)
**icon: 🎨**
```
不从语法开始，从"我想做什么"开始
- 例子：想做一个Harry Potter测验 → 直接开始设计界面
- 语法问题？交给AI解决
- 重点：保持创作热情
```

#### 原则 2: AI是队友 (AI as Co-Pilot)
**icon: 🤖**
```
学习如何"指挥"AI，而不是"成为"AI
- 学会提问：How do I...? Can you help me...?
- 学会调试：The error says... What does it mean?
- 学会验证：Does this code follow best practices?
```

#### 原则 3: 项目有灵魂 (Projects with Soul)
**icon: ✨**
```
每个项目都要回答：这是为谁做的？为什么做？
- 不做练习题，做真实作品
- 例子：给爸爸的生日卡片、Harry Potter粉丝测验
- 作品完成后：部署上线、分享给真实用户
```

### 3. 课程框架 (Teaching Framework)

#### 阶段 1: 探索期 (5-8岁)
**目标：建立"我能造东西"的自信**

| 周次 | 主题 | 工具 | 交付物 |
|------|------|------|--------|
| 1-2 | HTML基础 | Claude Code / Cursor | 个人名片页面 |
| 3-4 | CSS样式 | Lovable.dev | 涂鸦艺术页 |
| 5-6 | 交互初体验 | 简单按钮/弹窗 | 点击游戏 |
| 7-8 | 第一个完整项目 | Vercel部署 | 家庭相册网站 |

**教学方法：**
- 90% 时间动手，10% 讲解概念
- 使用可视化工具（Lovable、Webflow）
- 每节课必须有"成品"

#### 阶段 2: 建设期 (9-12岁)
**目标：理解"为什么"和"怎么做得更好"**

| 周次 | 主题 | 工具 | 交付物 |
|------|------|------|--------|
| 1-3 | React基础 | Vite + React | 待办事项应用 |
| 4-6 | 数据交互 | API调用 | 天气查询应用 |
| 7-9 | AI集成 | Claude API | AI聊天机器人 |
| 10-12 | 综合项目 | 全栈工具 | 个人作品集网站 |

**教学方法：**
- 引入"为什么这样设计？"的讨论
- 开始code review：AI写的代码好吗？
- 培养debug思维：错误是学习机会

#### 阶段 3: 创造期 (13-18岁)
**目标：独立创作有价值的产品**

| 项目周期 | 类型 | 示例 |
|----------|------|------|
| 4周项目 | SaaS工具 | 学习追踪器 |
| 6周项目 | 社交应用 | 校园活动平台 |
| 8周项目 | 创意实验 | AI艺术生成器 |

**教学方法：**
- 项目管理：从需求分析到上线
- 用户研究：谁会用？他们需要什么？
- 迭代思维：V1不完美，但要能用

### 6. 常见问题 (FAQ)

#### Q: 孩子需要先学Python吗？
**A:** 不需要。我们直接从web开发开始，因为：
- 即时可见的成果（打开浏览器就能看到）
- 更有创意表达空间（颜色、布局、动画）
- AI对web开发支持最好

#### Q: 如何处理挫败感？
**A:** Vibe Coding的核心就是降低挫败：
- 错误？问AI："这个错误是什么意思？"
- 不会？问AI："怎么做一个按钮？"
- 卡住？换个角度："我们先做界面，逻辑晚点再说"

#### Q: 会不会太依赖AI？
**A:** 这是特性，不是bug：
- AI是助手，决策仍然是孩子做的
- 随着能力提升，会逐渐理解AI生成的代码
- 最终目标：能判断AI的建议是否合理

#### Q: 如何评估学习效果？
**A:** 看作品，不看分数：
- 孩子是否能独立完成项目构思？
- 遇到问题是否知道如何求助（AI/导师）？
- 作品是否有个人风格和创意？

### 7. 课堂运营 (How We Actually Do It)

#### 典型60分钟课程流程

**0-10分钟: Check-in + 灵感分享**
```
- 上周项目有没有给朋友看？反馈如何？
- 本周想做什么？有什么新想法？
- （导师）分享一个酷炫案例激发灵感
```

**10-15分钟: 今日目标设定**
```
- 一起定义今天要完成什么
- 画草图/写下关键功能点
- 确保目标是"60分钟可完成的"
```

**15-50分钟: 实战开发**
```
学生主导：
- 尝试用AI实现功能
- 遇到问题先自己debug
- 卡住超过3分钟就问导师

导师角色：
- 引导，不直接给答案
- "你觉得问题可能在哪里？"
- "试试问AI：XXX"
```

**50-60分钟: 展示 + 反思**
```
- 演示今天做的东西（哪怕没完成）
- 讨论：哪里顺利？哪里卡住了？
- 下周目标预告
```

#### 家长参与指南

**不要做：**
- ❌ "这个代码不对，应该这样写..."
- ❌ "你要先学好数学才能编程"
- ❌ 拿孩子和别人比较

**要做：**
- ✅ "给我演示一下你做的东西？"
- ✅ "你是怎么想到这个点子的？"
- ✅ 帮忙分享作品到社交媒体

### 8. 导师培训 (For Teachers)

#### 成为Vibe Coding导师的三个心法

**1. 放下"标准答案"思维**
```
传统: "变量命名要用驼峰法"
Vibe: "这个名字让你想起它的作用吗？那就行"

重点：培养创造力 > 遵守规范
（规范可以后期refactor）
```

**2. 学会"苏格拉底式提问"**
```
学生: "我不知道怎么做"
导师: "如果你知道怎么做，你会从哪里开始？"

学生: "AI给的代码不工作"
导师: "错误信息说了什么？你觉得它在告诉我们什么？"
```

**3. 把AI当助教**
```
你的角色不是"知道所有答案"
你的角色是"知道如何找到答案"

示范如何：
- 有效地向AI提问
- 验证AI的回答
- 在AI基础上创新
```

## 页面技术实现建议

### 交互元素
1. **可展开的案例卡片** - 点击查看完整项目过程
